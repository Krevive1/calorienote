# パチスロ「朝一リセット狙い」初心者向けサイト構築ガイド

このドキュメントは、**「初心者でも見て実践しやすい情報サイト」**を GitHub で立ち上げ、将来的に広告収益化するための実行計画です。

---

## 1. ゴール設定（最初に固定する）

### 最終ゴール
- 広告収益を得る（月次で安定して黒字）

### 中間ゴール
- 3か月以内に「初心者向けの基礎記事」20本
- 6か月以内に「機種別リセット恩恵データ」50機種
- 主要検索キーワードで上位表示（例: 「機種名 リセット 恩恵」）

---

## 2. サイトの中身（初心者が迷わない構成）

### 必須ページ
1. **トップページ**
   - サイトの目的
   - 初心者導線（「まずここから」ボタン）

2. **はじめてガイド**
   - リセットとは何か
   - 天井短縮・モード優遇など恩恵の種類
   - 期待値の考え方（超やさしく）

3. **機種データベース**
   - 機種名
   - リセット時の恩恵
   - 狙い目ゲーム数
   - ヤメ時
   - 注意点（ホール差）

4. **立ち回り手順ページ**
   - 朝の入店前準備
   - 入店後の台選び
   - 実践時の判断フロー（図解）

5. **用語集**
   - 天井 / モード / 区間 / 据え置き など

6. **収支管理テンプレート配布**
   - Googleスプレッドシート等で無料配布

### UXの重要ポイント
- 1ページ1テーマ
- 先に結論（狙い目）→ 次に理由
- 難しい単語はリンクで補足
- すべてスマホ最適化

---

## 3. GitHubでの作り方（最短）

### 推奨構成
- **GitHub リポジトリ**: `slot-reset-guide`
- **公開方法**: GitHub Pages
- **サイト技術**: はじめは静的サイト（HTML/CSS/JS）で十分

### 初期ファイル例
- `index.html`（トップ）
- `beginner.html`（初心者ガイド）
- `machines/`（機種別ページ）
- `styles.css`
- `disclaimer.html`（免責事項）
- `privacy.html`（プライバシーポリシー）

### 公開手順（概要）
1. GitHubで新規リポジトリを作成
2. ファイルをpush
3. Settings → Pages で `main` ブランチを公開指定
4. 表示確認（URLが発行される）

---

## 4. 収益化までのロードマップ

### フェーズ1（0〜2か月）
- まずは広告なしで記事を蓄積
- 初心者向け記事を優先
- 機種ページを継続追加

### フェーズ2（3〜6か月）
- アクセス分析導入（GA4 / Search Console）
- 伸びる記事をリライト
- 内部リンク最適化

### フェーズ3（6か月〜）
- Google AdSense申請
- 収益記事（比較記事・実践データ）を強化
- 回遊率改善（関連記事導線）

---

## 5. SEO戦略（初心者向け特化）

### 狙うキーワード例
- 「パチスロ リセット 恩恵」
- 「機種名 リセット 天井短縮」
- 「朝一 狙い目 スロット」
- 「据え置き 見抜き方」

### 記事テンプレ（毎回同じ型にする）
1. 結論（狙えるか）
2. リセット恩恵
3. 狙い目/ヤメ時
4. 期待値イメージ
5. 注意点
6. 初心者向けQ&A

---

## 6. 信頼性を上げる運用ルール

- 情報の更新日を明記
- 参考元（メーカー・解析サイト・実戦データ）を明記
- 不確定情報は「推測」と明確化
- 誤情報は修正履歴を残す

---

## 7. 法務・コンプライアンス（重要）

- 20歳未満の遊技を助長する表現は避ける
- 「必ず勝てる」等の断定表現を使わない
- 依存を煽る表現を避ける
- 免責事項ページを必ず設置
- 広告掲載時はプライバシーポリシー（Cookie/広告配信）を整備
